Create a Server

As mentioned previously, Vonage will bridge the communication between the phone call and our Agent through Webhooks.

With the setup script that ran at the beginning of the workshop to create the Vonage application and purchase a number, it also set up the Webhook endpoints that we used.

Here, we will actually create the Webhook routes that the Vonage application is expecting.

Flue comes with a way to do this routing right out of the box. By creating the optional src/app.ts, we can create the custom Webhook routes that Vonage needs by spinning up a Hono server.

When Vonage sends a request, it is expecting a response that will tell it what to do with the call. These instructions are packaged in a Network Call Control Object (NCCO).

There are a lot of different actions that can be used during a call, but for this workshop, we will be using:

Talk: so that the Agent can speak.

Input: so that we can process what the caller is saying.

Stream: to keep the call open, we’ll play music while the AI Agent thinks of a response.
