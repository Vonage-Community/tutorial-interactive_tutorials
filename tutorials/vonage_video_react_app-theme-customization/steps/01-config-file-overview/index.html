Design Tokens Overview

All visual styles for the Vonage Video React App are driven by a single file at the project root: designTokens.json. This is the one place you need to go to control colors, typography, and shape.

Where It Lives

Section titled “Where It Lives”
  • designTokens.json ← All theme configuration goes here
  • designTokens.example.json ← Auto-generated reference (do not edit)
  • Directorybackend/
    • …
  • Directoryfrontend/
    • …
  • Directorylibs/
    • Directoryui/
      • Directorysrc/
        • Directorytheme/
          • Directoryhelpers/
            • DirectorydesignTokens/
              • designTokens.json ← Synced automatically from root

Caution

Never edit libs/ui/src/theme/helpers/designTokens/designTokens.json directly. It is overwritten every time you run yarn sync:theme-tokens. Always edit the root designTokens.json.

File Structure

Section titled “File Structure”

designTokens.json has three top-level sections:

  • light / dark — Separate color palettes for each mode. Both modes use the same set of semantic token keys.
  • Typography tokens — Sit at the top level (outside light/dark) and apply to both modes. Includes fontFamilyPlain and named scales like headline, bodyBase, caption.
  • Shape tokens — Also at the top level. Six borderRadius* values control the roundness of buttons, cards, and dialogs.

Key color tokens

Section titled “Key color tokens”
TokenRole
primaryBrand accent — buttons, active states
primaryHoverHover state of the primary color
surfaceCard and panel backgrounds
backgroundPage background
textPrimaryPrimary text color
textSecondarySecondary / subdued text
errorDestructive actions and error states
successConfirmation and success states
